# Project governance

SBS uses a maintainer-led, community-informed model.

- The project maintainer sets product direction, release timing and security decisions.
- Contributors propose changes through issues and pull requests.
- Financial correctness, data protection and backward compatibility take priority over convenience.
- Material changes to the budget format, permissions or calculation semantics require documentation, tests and maintainer approval.
- Decisions with broad impact should be recorded in a public issue or pull request.

The maintainers may reject changes that introduce legal uncertainty, proprietary data, unsafe authentication patterns or maintenance costs disproportionate to community benefit.

Contributions are accepted under AGPL-3.0 using Developer Certificate of Origin sign-off. The software license does not grant rights to the SBS or Scognamiglio Budgeting System names or branding.
